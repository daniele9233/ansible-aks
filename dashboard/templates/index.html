Vault

Decrypt and inspect Ansible vault files. Read-only.

Required to decrypt vault.yml and vault_cavalid.yml.

vault.yml
group_vars/all/vault.yml

Master Ansible vault containing cluster secrets and SSH keys.

vault_cavalid.yml
group_vars/all/vault_cavalid.yml

CA-valid certificate bundle for the trusted Rancher deployment.

Dry-Run

Run the playbook in --check --diff mode. Nothing is applied to the cluster.

tag: rancher_ca_valid
CA-Valid mode

Simulate Rancher install with corporate CA certificates (trusted chain).

tag: rancher_self_signed
Self-Signed mode

Simulate Rancher install generating its own self-signed cert.

Dry-run reports changes that would be applied. Use this to validate vars and tags before a real deploy.

Deploy

Apply the Rancher playbook to the live AKS cluster. Changes are real.

This will install / upgrade Rancher on the production cluster.
Deploy · CA-Valid

Install Rancher with corporate CA-signed certificates.

Deploy · Self-Signed

Install Rancher with its bundled self-signed certificate.

Uninstall

Remove Rancher from the AKS cluster. Destructive action — confirmation required.

This will run bash uninstall-rancher.sh

All Rancher resources, namespaces and CRDs will be removed. Workloads outside the Rancher namespace are not touched. No vault password required.

Windows Tunnel

SSH tunnel + kubectl port-forward commands for accessing the Rancher UI from a Windows workstation.

Aggiungi questa riga al file hosts di Windows

In modalità CA Valid il nome host deve risolvere a localhost. Modifica C:\Windows\System32\drivers\etc\hosts come amministratore.

127.0.0.1  rancher.app.iac-svil.almaviva.it

SSH Commands

Tre flussi di accesso: Rancher (CA Valid / Self-Signed) e Dashboard Ansible. Esegui i comandi nell'ordine indicato.

Rancher · CA Valid

Accesso via Traefik con certificato wildcard AlmavivA (lucchetto verde).

cert valido
1. SSH tunnel · Windows PowerShell (lascia aperto)
ssh -i "C:\Users\D.Pascolini\Desktop\IAC-Azure\ALMDEPIAC01LTAZ_key 1.pem" -L 8443:localhost:8443 azureuser@10.207.201.136
2. kubectl port-forward · dentro la sessione SSH della VM
kubectl port-forward -n traefik svc/traefik 8443:443
3. Browser su Windows
https://rancher.app.iac-svil.almaviva.it:8443
Prerequisito · file hosts Windows

Aggiungi questa riga in C:\Windows\System32\drivers\etc\hosts (come Amministratore):

127.0.0.1  rancher.app.iac-svil.almaviva.it

Rancher · Self-Signed

Accesso diretto al servizio Rancher (TLS gestito da dynamiclistener).

self-signed
1. SSH tunnel · Windows PowerShell (lascia aperto)
ssh -i "C:\Users\D.Pascolini\Desktop\IAC-Azure\ALMDEPIAC01LTAZ_key 1.pem" -L 8443:localhost:8443 azureuser@10.207.201.136
2. kubectl port-forward · dentro la sessione SSH della VM
kubectl port-forward -n cattle-system svc/rancher 8443:443
3. Browser su Windows
https://localhost:8443/login
Avviso certificato

Il browser mostrerà un warning di certificato non valido (atteso in modalità self-signed). Clicca Avanzate → Procedi comunque a localhost.
